    About

    Welcome! This is the web version of my notes, including only the mostly polished[1] parts of them. It is built using Gerby, created by Pieter Belmans and Raymond Cheng.

    Disabled Features. This website is hosted via GitHub pages, and is thus completely static, in contrast to the Stacks project, Kerodon, and the Automorphic project. As such, certain features found in those projects are disabled for this site, including the search function and comments.

    Comments. Although the comments feature has been disable for this site, any and all comments, no matter how small (even things like bad typography or very small typos), are extremely welcome, as are contributions and feedback. If you'd like to get in touch, you can reach me at emily.de.oliveira.santos.tmf@gmail.com.

    PDF Styles. The PDFs for each chapter as well as for the whole book are generated using four different styles, as summarised in the following table:

    Style Class Font Theorem Environments
    Style 1 book Alegreya Sans tcbthm
    Style 2 book Alegreya Sans amsthm
    Style 3 book Arno* amsthm
    Style 4 stacks-project.cls Computer Modern amsthm

    *To be replaced with Linus Romer's Elemaints when it is released.

    Footnotes. There are lots of footnotes, due to how the original PDFs were created. They automatically expand when you hover your mouse over them; here's an example[2].

    Large Diagrams and the Zoom In Feature. This work features many diagrams that are unfortunately too large to be comfortably legible in their native size. To compensate for this, it is possible to click on them to expand their size by 200%, or alternatively open them as separate images in new tabs by right-clicking and then selecting “Open image in new tab”.

    Placeholder Symbols. Currently, a number of macros have been defined using placeholder symbols, and look very ugly as a result. This is because I eventually plan to replace them using a custom font (WIP).

    Warning: These notes are quite informal and most likely contain lots of errors, so please use them at your own risk. They are very idiosyncratic and use lots of very nonstandard terminology, so please keep this in mind as well.

    Footnotes

    [1] The name “The Gigantic Mess Project” is a joke; I started calling my notes this because they originally started as Anki cards, and have always been a complete mess since.
    [2] Most of the footnotes are quite unimportant and can be ignored, though.
